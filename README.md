# L-Asset 轻量级资产管理系统

> **Snipe-IT 很好用，但是不能即插即用，部署又有门槛。**
> **L-Asset 适合中小企业，轻便，快捷，精准。**

不需要 Docker、不需要 PHP、不需要 MySQL、不需要 Composer。
一个 exe 文件 + 双击运行 = 开箱即用。

## 🎯 设计理念

Snipe-IT 是一个优秀的资产管理系统，但它的缺点在中小企业场景中很明显：

| 对比项 | Snipe-IT | L-Asset |
|--------|----------|---------|
| 部署方式 | Docker/PHP+MySQL | **双击运行，零配置** |
| 依赖 | PHP、MySQL/ MariaDB、Composer、Node.js | **零依赖，单文件** |
| 数据库 | 需要单独部署 MySQL | **自带 SQLite，即开即用** |
| 升级 | composer update + migrate | **替换 exe 文件即可** |
| 备份 | 需备份 MySQL + 文件 | **一个 .db 文件搞定** |
| 跨设备迁移 | 重装环境很麻烦 | **导出 XML → 导入 XML，一键还原** |

**L-Asset 的目标：** 让 IT 管理员在 30 秒内部署好资产管理系统，把时间花在管资产上，而不是管系统上。

## ✨ 功能特性

- 🪶 **超轻量**：单二进制文件 + SQLite 数据库，零外部依赖
- 🎨 **美观 Web 界面**：响应式设计，支持 PC/手机使用
- 🧩 **自定义字段**：自由扩展资产信息字段，无需改代码
- 📦 **CSV 批量导入/导出**：支持从 Excel 导入已有资产
- 🔍 **搜索筛选 + 分页**：支持关键词搜索、状态/类型筛选、每页条数切换
- 📸 **附件上传**：支持上传照片、PDF，每个资产可挂多个文件
- 🖨️ **打印标签**：批量打印资产标签，扫码可直接访问详情页
- 📋 **全生命周期管理**：入库 → 领用 → 归还 → 报废 → 复活，全流程操作记录审计
- 🗑️ **报废/复活机制**：报废需填写原因（损坏/被盗/淘汰），资产移至独立报废库；支持从报废列表一键复活并填写理由
- 📄 **报废资产独立列表**：单独页面管理报废资产，支持搜索、分页、详情查看
- 🔐 **多用户 + 权限控制**：管理员/普通用户两种角色
- ☁️ **ZIP 完整打包备份**：一键导出/导入（含附件图片和PDF），换设备迁移无压力
- 🔗 **URL 状态保持**：筛选条件同步到 URL，刷新不丢失
- 🎨 **主题切换**：暖橙 / 淡蓝 两套马卡龙色系，导航栏一键切换
- 💰 **财务配置**：税率 + 汇率换算，资产价格自动计算含税/不含税/多币种
- 📍 **可管理预设字段**：位置、保修年限等字段可从设置页添加/删除预设值
- 📅 **保修自动计算**：选保修年限 + 采购日期 → 自动推算保修到期日
- 🔄 **主题色 Logo**：SVG 设备图标随主题自动变色
- 📋 **智能粘贴**：粘贴设备信息文字，自动提取品牌/型号/SN/CPU/内存/硬盘等，一键填入

## 🚀 快速开始（Windows）

### 方式一：双击运行（推荐）

1. 下载 `output/l-asset.exe`
2. 双击运行
3. 浏览器打开 `http://localhost:5678`
4. 默认管理员：`admin` / `admin123`

> 所有数据保存在 exe 同目录下的 `data/l-asset.db` 文件中。

### 方式二：自定义配置

创建快捷方式，目标设为：

```
D:\l-asset\l-asset.exe
```

如需修改端口或数据目录，创建 `start.bat`：

```batch
@echo off
set LASSET_PORT=8080
set LASSET_DATA=D:\l-asset-data
l-asset.exe
pause
```

## 🚀 快速开始（Linux / macOS / NAS）

### 使用 Go 源码编译（推荐）

```bash
# 需要 Go 1.21+
git clone <仓库地址>
cd l-asset

# 编译
go build -o l-asset .

# 运行（默认 5678 端口）
./l-asset

# 自定义端口
LASSET_PORT=3000 ./l-asset

# 自定义数据目录
LASSET_DATA=/path/to/data ./l-asset
```

### 使用预编译二进制（如果提供了 Release）

```bash
# 下载对应平台的可执行文件
wget https://.../l-asset-linux-amd64
chmod +x l-asset-linux-amd64
./l-asset-linux-amd64
```

### NAS 部署（以飞牛 fnOS / 群晖为例）

```bash
# SSH 登录到 NAS
# 上传二进制到 /volume1/docker/l-asset/
# 直接运行
cd /volume1/docker/l-asset
./l-asset &

# 设置开机自启（通过 NAS 的任务计划）
# 或用 nohup + screen 管理
```

> **提示：在 NAS 上长期运行时，建议使用 screen 或 tmux：**
> ```bash
> screen -S lasset
> ./l-asset
> # Ctrl+A+D 断开，screen -r lasset 重连
> ```

## 🔧 部署教程

### 第一步：选择运行方式

| 场景 | 推荐方式 |
|------|----------|
| IT 小白，只想用 | Windows 双击 exe |
| 公司内网服务器 | Windows 开机自启 |
| NAS 长期运行 | Linux 编译 + nohup |
| 技术团队 | Docker 映射端口 + 数据卷 |

### 第二步：访问系统

浏览器打开 `http://<服务器IP>:5678`

- 同一局域网所有设备均可访问
- 如果需要外网访问，配置路由器端口转发（**注意安全！建议配合 VPN 使用**）

### 第三步：初始设置

1. 用 `admin` / `admin123` 登录
2. 进入 **设置** → 修改企业名称（用于资产编号前缀）
3. 进入 **设置** → 设置税率和汇率（如果有外币需求）
4. 进入 **设置** → 修改管理员密码
5. 进入 **用户管理** → 添加使用者账号
6. 进入 **字段管理** → 按需添加自定义字段、位置、保修年限的预设值

## 📖 使用指南

### 概览页

仪表盘显示资产总量、在库数、已领用数、报废数、笔记本/台式机统计。

### 资产管理

#### 添加资产
- 点资产列表页右上角 **"+ 添加资产"**
- 填写信息，资产编号留空会自动生成（格式：企业名称-序号）
- 添加时支持搜索使用人

#### 编辑资产（新！）
- 在详情页点右上角 **"编辑"** 按钮
- 所有字段直接变成可编辑状态
- 修改完点 **"保存修改"** 即刻生效
- 自定义字段同步可编辑

#### 克隆资产（新！）
- 在资产列表每行的操作区点 **"克隆"**
- 自动将原资产信息填入添加表单
- 修改编号/序列号等即可快速添加同类设备

#### 领用 / 归还 / 报废
- 在资产详情页右侧操作区执行
- 可选多人时支持搜索
- 所有操作自动记录到操作日志

#### 批量操作（新！）
- 资产列表勾选多个资产（前面打勾）
- 顶部工具栏出现批量操作按钮
- **批量领用**：选人 → 一次领用全部
- **批量归还**：一键归还全部
- **批量报废**：确认后报废全部
- **批量删除**：需手动输入"确认删除"验证

#### 打印标签（新！）
- 详情页或列表批量勾选 → 点 **"🖨️ 打印标签"**
- 新窗口显示标签，包含资产编号、类型、扫码链接
- 浏览器打印 → 贴到设备上

#### 附件/照片（新！）
- 在资产详情页 **"附件/照片"** 区域
- 点右上角 **"上传"** 选择图片或 PDF
- 图片显示缩略图，点击可查看原图
- 支持格式：jpg/png/gif/webp/bmp/pdf（单个文件≤5MB）

### 搜索与筛选

| 功能 | 说明 |
|------|------|
| 关键词搜索 | 匹配资产编号、品牌、型号、序列号、使用人，匹配文字高亮 |
| 状态筛选 | 在库 / 已领用 / 已报废 |
| 类型筛选 | 笔记本 / 台式机 等 |
| 每页条数 | 20 / 50 / 100 / 200 可选 |
| URL 保持 | 搜索和筛选条件自动同步到浏览器地址栏，刷新不丢失 |

### 用户管理

- 管理员可添加/编辑/删除用户
- 用户角色分 **管理员** 和 **普通用户**
- 普通用户只能查看资产，不能进入设置/字段/导入页面
- 创建用户时密码留空默认为 `123`

### 操作记录

所有入库、领用、归还、报废、编辑操作都有详细审计日志。

## 💾 数据备份与迁移

### 方式一：ZIP 完整打包备份（推荐，含附件图片和PDF）

进入 **设置 → 系统备份**，点击 **📦 导出完整备份**，下载一个 ZIP 文件。
包含：资产数据、自定义字段、用户、操作记录、配置、**附件图片和PDF**。

导入时点 **导入恢复**，选择 ZIP 文件即可。

### 方式二：XML 仅数据备份（不含附件）

进入 **设置 → 系统备份**，点 **仅备份数据** 区下方的 **导出 XML**。
包含：资产数据、自定义字段、用户、操作记录、配置。

### 方式三：直接复制数据库文件

```
l-asset/
└── data/
    └── l-asset.db     ← 复制这个文件即可
```

把 `.db` 文件复制到新设备同目录下启动即可。

### 方式四：CSV 导出（仅资产数据）

进入 **设置 → 数据导出（CSV）**，下载所有资产为 CSV，可用 Excel 打开编辑后再导入。

### 跨设备迁移步骤（推荐）

```
旧设备 → 设置 → 导出完整备份（ZIP）
新设备 → 运行 → 设置 → 导入恢复 → 选择 ZIP → 完成
```

## 🔒 安全注意事项

- 默认管理员密码是 `admin123`，**首次部署后请立即修改**
- 上传文件限制：仅图片（jpg/png/gif/webp/bmp）和 PDF，单文件 ≤ 5MB
- 文件类型经过魔数验证，改后缀名无法绕过
- 上传目录有登录鉴权
- 批量删除需手动输入"确认删除"验证，防止误操作
- 不建议将系统直接暴露到公网，建议通过 VPN 或内网访问

## ❓ 常见问题

### Q: 忘记管理员密码了？
直接操作数据库重置：
```bash
# 使用 sqlite3 命令行
sqlite3 data/l-asset.db
UPDATE users SET password='240be518fabd2724ddb6f04eeb1da5967448d7e831c08c8fa822809f74c720a9' WHERE name='admin';
.quit
# 密码重置为 admin123
```

### Q: 端口被占用了？
设置环境变量 `LASSET_PORT=8080` 使用其他端口。

### Q: 数据库在哪？
默认在 exe 同目录下的 `data/l-asset.db`。

### Q: 如何升级？
替换 exe 文件，重新启动即可。数据库向后兼容。

### Q: 支持外网访问吗？
配置路由器端口转发即可。但**强烈建议配合 VPN（如 Tailscale/WireGuard）使用**，不要直接暴露到公网。

### Q: CSV 导入支持哪些列名？
支持中文和英文两种列名：
- 中文：资产编号、类型、品牌、型号、序列号、使用人……
- 英文：asset_tag、type、brand、model、serial、current_user……
- 自定义字段：列名与自定义字段管理中定义的名字一致即可
