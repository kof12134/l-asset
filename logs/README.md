# L-Asset 项目日志读取指南

项目日志位于 `logs/CHANGELOG.md`，记录了从项目建立到当前的全部重要修改。

## 文件位置

```
/vol1/1000/WorkSpace/asset-manager/logs/CHANGELOG.md
```

## 日志内容

- 功能变更与原因
- 遇到的关键问题和解决思路
- 架构决策及其理由
- 数据库表结构
- API 列表
- 编译部署方式
- 已知问题和待办

## 目的

防止 AI 失忆后无法理解项目演进历史。日志以人类可读方式记录了「为什么这么做」而非仅仅「做了什么」。

## 更新频率

建议每次新增功能或修复 Bug 时更新。
